Exercise ‹12›:

DOMINATING SET ≤\leq≤ SAT
Reduce the DOMINATING SET problem to the SAT problem. These problems are defined as follows:
  • DOMINATING SET: given a natural kkk and an undirected graph GGG, is there a subset SSS of size at most kkk of nodes that dominates GGG? In other words, is there a subset SSS of nodes with size at most kkk such that any node of GGG is either in SSS or adjacent to a node in SSS?
    More formally, this problem can be defined as the following set:
    {⟨k,G=⟨V,E⟩⟩∣∃S⊆V:(∣S∣≤k  ∧  ∀u∈V:(u∈S  ∨  ∃v∈S:{u,v}∈E))}\{ \langle k,G=\langle V,E\rangle\rangle \mid \exists S\subseteq V: (|S|\leq k\;\wedge\;\forall u\in V: (u\in S\;\vee\;\exists v\in S: \{u,v\}\in E)) \}{⟨k,G=⟨V,E⟩⟩∣∃S⊆V:(∣S∣≤k∧∀u∈V:(u∈S∨∃v∈S:{u,v}∈E))}
  • SAT: given a boolean formula FFF in conjunctive normal form, is FFF satisfiable?
    More formally, this problem can be defined as the following set:
    {F=C1∧…∧Cn∣F satisfiable}\{ F=C_1\wedge\ldots\wedge C_n \mid F\text{ satisfiable} \}{F=C1​∧…∧Cn​∣F satisfiable}
The input and output of the reduction conform to the following data types: 
  • in: struct {
      k: int
      numnodes: int
      edges: array of array [2] of int
      adjacents: array of array of int
      adjacencymatrix: array of array of int
    }
    The input is a natural number in.k and an undirected graph. The graph is given by means of four different kinds of information: the number of nodes in.numnodes, the list of edges in.edges, for each node the list of its adjacent nodes in.adjacents, and the adjacency matrix in.adjacencymatrix. The nodes are integers between 111 and in.numnodes, and thus, the 000’th position of in.adjacents does not contain useful data and should be ignored; for any other position ppp, in.adjacents[p] is the list of nodes adjacent to ppp. For the same reason, the 000’th row and column of in.adjacencymatrix do not contain useful data and should be ignored; for any other row rrr and column ccc, in.adjacencymatrix[r][c] is 000 when there is no edge between the nodes rrr and ccc, and 111 otherwise.

    Note: The input graph has at least one node, no repeated edges, nor self-loop edges, and the value of in.k is at most the number of nodes of the graph. 
  • out: array of array of string
    The output is a list of clauses. A positive literal is represented by a positive number or by a string not starting with «-», a negated literal by a negative number or a string starting with «-».

    Note: If the output is an empty formula, it is considered satisfiable. If the formula has any empty clause, it is considered unsatisfiable.
