Introducci

En un estudi per estudiar el nivell de colesterol en sang, en persones en edat de creixement (9-20 anys), sha plantejat un model per explicar el nivell de colesterol (C) en funci del pes (P), edat (E) i alada (H).

s conegut que el pes s un dels factors que influeixen en laugment del nivell de colesterol

A lenunciat hi ha una falta de precisi, no s el pes el que fa augmentar el colesterol, si no que s lexcs de pes. Per definir lexcs de pes, EP, la informaci que tenim s que EP = max{0, 0.5H - 10}

En aquest script es veuen els segents conceptes:

  • Model lineal general com cas particular del model lineal generalitzat
  • Com fer la validaci del model lineal generalitzat
  • Com fer prediccions amb bandes de predicci
  • Com afecta la funci link i la familia a lajust

Descriptiva prvia

Colesterol en funci de lexcs de pes ajustat per ledat

scatterplot(C ~ EP|E,smooth=F,boxplots=F,col=1:12,dd)

Colesterol en funci de ledat

scatterplot(C ~ E,   smooth=F,boxplots=F,col=1:12,dd[dd$EP==0,]) # Els que no tenen exces de pes

Model lineal general com cas particular del model lineal generalitzat

En aquest apartat veurem com el Model lineal general (MLG) s un cas particular del model lineal generalitzat (MGLz) i que tot el que obtenim dun s igual a laltre. Comenarem ajustant el model nul i el model amb E i EP emprant lm (MLG) i glm (MLGz).

# Models nuls
mod0  <-  lm(C ~ 1, dd)
modg0 <- glm(C ~ 1, dd, family = gaussian(link=identity))

# Models amb exces de pes i edat
mod   <-  lm(C ~ EP + E, dd)
modg  <- glm(C ~ EP + E, dd, family = gaussian(link=identity))

Altres funcions link

Com el que falla s la tendncia central, hem de provar altre funcions links.

modg_inv  <- glm(C ~ EP + E, dd, family = gaussian(link="inverse"))
modg_inv2 <- glm(C ~ EP + E, dd, family = gaussian(link="1/mu^2"))
modg_log  <- glm(C ~ EP + E, dd, family = gaussian(link="log"))

Veiem el grfic de rsidus versus valors predits dels 4 links.

par(mfrow=c(2,2))
residualPlot(modg ,    main='Identity')
residualPlot(modg_inv, main='Inverse')
residualPlot(modg_inv2,main='Inverse^2')
residualPlot(modg_log, main='Log')

Sembla que la funci link log, millora una mica la tendencia central. Comparem les prediccions del model amb link log i link identitat.

##-- Valors a predir
E   <- 9:20
EP  <- 0
df  <- data.frame(E=E, EP=EP)
dd0 <- dd[dd$EP==0,]

##-- Prediccions link identitat
par(mfrow=c(1,2))
PP    <- predict(modg, df , type = "response")
disp  <- summary(modg)$dispersion
V     <- 1
ylim  <- range(PP) + c(-2,2) * sqrt(V*disp)
plot(C ~ E, dd0, ylim=ylim, main="Prediccions (EP=0) link identitat")
lines(E, PP)
lines(E, PP + 2 * sqrt(V*disp), col="red",lty=2)
lines(E, PP - 2 * sqrt(V*disp), col="red",lty=2)

##-- Prediccions link log
PP    <- predict(modg_log, df , type = "response")
disp  <- summary(modg_log)$dispersion
V     <- 1
ylim  <- range(PP) + c(-2,2) * sqrt(V*disp)
plot(C ~ E, dd0, ylim=ylim, main="Prediccions (EP=0) link log")
lines(E, PP)
lines(E, PP + 2 * sqrt(V*disp), col="green",lty=2)
lines(E, PP - 2 * sqrt(V*disp), col="green",lty=2)

Altres families

La familia no lhauriem de ajustar perqu sembla que no hi ha problemes de heteroscedasticitat. Igualment fem proves per veure com influeix. Mentenim el link log

modg_nor <- glm(C ~ EP + E, dd, family = gaussian(link="log"))
modg_gam <- glm(C ~ EP + E, dd, family = Gamma(link="log"))
modg_iga <- glm(C ~ EP + E, dd, family = inverse.gaussian(link="log"))

Veiem el grfic de residus versus valors predits de 3 families.

par(mfrow=c(2,2))
residualPlot(modg_nor, main='Normal')
residualPlot(modg_gam, main='Gamma')
residualPlot(modg_iga, main='Inverse Gaussian')

Amb la familia Normal sembla que va millor. Comparem prediccions amb Gamma
